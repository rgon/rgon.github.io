TraktorLCD

An easy-to-use Traktor display application.

It allows DJs to use their controllers in conjunction with their phones to achieve a seamless CDJ-like experience.
TraktorLCD connects with your (Linux only) computer automatically via a WiFi hotspot (no internet required).

All your track info is displayed on your screen, as well as the track's key and BPM, waveform and a live VU meter.

All this in the extremely portable package that your phone is.


Estimated release date: Summer 2017

Programming Languages: HTML5+JS (WebSockets), Python3, XML (Database)

Task Manager

An intuitive task manager and performance monitor for Linux.

Offering powerful features to advanced users as well as a simplified view for tinkerers, this Task Manager allow you to diagnose and improve your computer with an aesthetically pleasing user interface.

With nice features like GPU monitoring, PCI/USB/AUDIO device listing and diagnosing, temperature logging, multithreading percentages for application performance diagnosis.


Estimated release date: Summer 2017

Programming Languages: HTML5+JS, Python3, Bash (Linux)

Acerca de esta página: Proyecto de TIC

Al principio, esta página iba a ser una experiencia bastante interesante, con desplazamiento con efecto de paralelaje (acelerado por GPU), efectos al bajar por la página y bastantes ideas más.

Sin embargo, el código que tenía ya escrito no tenía la calidad suficiente. Mientras que todo se veía perfecto sin imagenes ni contenido, solo bloques de color, todo se desintegraba rápidamente al usar distintos navegadores. Esto hizo que el tiempo necesario para acabar esta página fuese cada vez más y más, hasta el punto en el que era ya demasiado tarde.

Por lo tanto, lo que hice fue coger uno de mis diseños antiguos (bueno, del verano de 2016) y escribir un contenido acorde con este trabajo.

Está escrita en HTML, sin utilizar ningun framework (sin Bootstrap ni similares), únicamente código escrito por mí (a excepción de jQuery para JavaScript y los efectos de scrolling, por no reinventar la rueda aún más).

¿Y qué será de esa idea de una página distinta? Desde luego, este dominio será actualizado cuando esté terminado. Esta idea requiere utilizar tecnologías de HTML/CSS "punteras" y no diseñadas para blogs sino para juegos. Algo problemático, por supuesto. Es probable que necesite crear un preprocesador que organice todo, y eso me va a llevar tiempo.

Y todo esto para que pueda ir fluido en un móvil o PC de gama baja...

Esta página sin embargo es notablemente más simple, aunque podría ser mejor. Y está llena de malas prácticas y chapuzas con el CSS.


Siento hablar en un lenguaje tan críptico, pero sin una demostración es difícil coger la idea. Aparte de ser un lío de idiomas, mezclando inglés y español. En fin...


PS: no es un blog, sino una página estática dando el pego.


Un saludo.
